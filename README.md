# Sentiment Analysis - Advanced Logistic Regression Edition

This project focuses on a robust text classification pipeline for Sentiment Analysis. It implements advanced natural language processing abstractions and an enhanced Logistic Regression model tuned for sparse textual data.

## Features & Improvements

1. **Contraction Expansion**: Expands contractions systematically (e.g., "don't" → "do not") to expose negation tokens.
2. **Negation Scope Marking**: Implements Pang et al. (2002) negation marking (e.g., "not good" → "NOT_good").
3. **Emphasis Normalization**: Normalizes capital letters and multi-punctuation (e.g., "GREAT!!!" → "CAPS_great MULTI_EXCLAIM").
4. **Elongation Handling**: Normalizes word elongations (e.g., "sooooo" → "so ELONGATED").
5. **Emoticon Tokenization**: Maps emoticons to sentiment tokens (`:)` → `EMOTE_POS`, `:(` → `EMOTE_NEG`).
6. **Advanced N-grams**: 
   - Words TF-IDF (1 to 3 grams) to capture multi-word sentiments (e.g., "not worth it").
   - Character TF-IDF (Word boundaries, 2-5 chars) to capture misspellings and morphological patterns.
7. **Transductive Vectorizer Fit**: Fits the vocabulary on both the train and test text (without label leakage) for better vocabulary mapping.
8. **Hand-crafted Features**: Engineer custom features like net sentiment score, negated POS/NEG counts, and capitalization metrics.
9. **Optimized Logistic Regression**: Utilizes a `OneVsRest` architecture with a `liblinear` solver for speed on highly sparse textual data. Hyperparameter tuned to `C=2.0`.
10. **Pseudo-labeling**: Augments the dataset with high-confidence predictions on the test set.

## Project Structure

- `SEntimentAnalysis.ipynb`: The core Jupyter Notebook containing data preprocessing, feature engineering, model building, cross-validation, and prediction generation.
- `train.csv`: The training dataset containing the text sentences and their corresponding sentiment labels.
- `test.csv`: The testing dataset used to evaluate and generate predictions on unknown labels.
- `submission.csv`: Generated by the notebook resulting from model predictions on the `test.csv` dataset.

## How to use

1. Install required dependencies listed in the main notebook (e.g., `scikit-learn`, `numpy`, `pandas`, `scipy`).
2. Run the `SEntimentAnalysis.ipynb` notebook from top to bottom.
3. The script will preprocess the text, train the pipeline, perform cross-validation, apply pseudo-labeling, and finally output the predictions into `submission.csv`.
